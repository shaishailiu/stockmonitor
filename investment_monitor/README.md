自动监控 A股 + 港股资产的底部投资机会，当触发预警条件时推送提醒。

## 📊 系统状态

- ✅ **数据源**：腾讯财经接口（稳定可靠）
- ✅ **监控范围**：17只资产（5只A股 + 12只港股）
- ✅ **定时任务**：每日15:30自动扫描

## 🚨 预警规则

### 🟡 黄色预警（进入观察）
- 回撤 ≥ 30% + RSI(14) < 35
- 回撤 ≥ 30% + 估值 < 历史30%分位

**操作建议：** 观察，暂不操作

### 🟠 橙色预警（持续关注）
- 回撤 ≥ 40% + RSI(14) < 30 + 周线RSI < 35
- 回撤 ≥ 40% + 估值 < 历史15%分位
- 回撤 ≥ 40% + 成交量 < 60日均量的50%

**操作建议：** 可小仓位试探（10%）

### 🔴 红色预警（重点关注）
- 回撤 ≥ 50% + 周线RSI < 30
- 回撤 ≥ 50% + 估值 < 历史10%分位
- 回撤 ≥ 50% + MACD周线底背离

**操作建议：** 可分批建仓（20-30%）

## 📅 定时任务

已配置两个定时任务：

1. **每日扫描**：每天15:30自动执行
   - 扫描所有资产
   - 计算回撤和技术指标
   - 触发预警时推送到企业微信群

2. **周报**：每周日20:00生成
   - 汇总本周所有预警
   - 分析回撤变化趋势
   - 推荐下周重点关注标的Top3

查看定时任务：
```bash
openclaw cron list
```

## 📈 数据说明

### 历史数据积累
- 首次运行时，使用当前价作为52周高点基准
- 之后每日自动保存数据到 `data/` 目录
- 随着时间推移，历史数据会越来越准确
- **建议：** 运行2-3周后，回撤计算会更加精确

### 技术指标
- **RSI(14)**：基于历史缓存计算，至少需要14天数据
- **52周高低点**：基于本地缓存数据（逐日积累）
- **估值指标（PE/PB）**：暂未实现，待后续集成

## ❓ 常见问题

**Q: 为什么美股暂不支持？**  
A: 腾讯内网环境对 Yahoo Finance 有限流，已测试多个方案均不可用。建议联系IT部门申请外网白名单。

**Q: 如何测试定时任务？**  
A: 运行 `openclaw cron run <jobId>` 可立即触发执行。

**Q: 52周高点不准确怎么办？**  
A: 首次运行会使用当前价作为基准，需要积累2-3周数据后才准确。可手动编辑 `data/` 中的历史数据。

**Q: 如何添加新的监控资产？**  
A: 编辑 `config.json`，在对应的 `a_stock` 或 `hk_stock` 数组中添加资产信息。
